# Sericia SEO / GEO / Indexing operator runbook

This runbook covers everything needed to take a freshly deployed
Sericia URL from "live" to "ranking on Google / Bing / Perplexity"
in the shortest possible window.

<a id="seo-1"></a>

## 1. Indexing strategy overview

The Sericia indexing stack runs on three parallel rails:

| Rail | What it indexes | Time-to-first-result |
|------|----------------|---------------------|
| **Google Search Console** | Google + Discover + News (ultimately ChatGPT/Gemini) | 24h–14 days |
| **IndexNow API** | Bing, Yandex, Naver, Seznam, Yep (also DuckDuckGo, Ecosia, AOL via Bing relay) | hours |
| **Sitemap auto-discovery** | All crawlers via `/robots.txt` + `/sitemap.xml` | days |

The total addressable URL surface as of F40:

| Route | Base URLs | × hreflang | Total surfaces |
|-------|-----------|-----------|----------------|
| `/` + `/products` + 5 hubs | 5 | 10 | 50 |
| `/products/[slug]` | ~10 | 10 | 100 |
| `/guides/[country]/[product]` | 144 | 10 | 1,440 |
| `/compare/[a]/[b]` | 66 | 10 | 660 |
| `/uses/[product]/[case]` | 72 | 10 | 720 |
| `/journal/[slug]` | ~10 | 10 | 100 |
| `/tools/*` | 8 | 10 | 80 |
| `/articles/[slug]` | ~5 | 10 | 50 |
| Static brand + legal | 12 | 10 | 120 |
| **Total** | **332** | | **≈ 3,320** |

## 2. Google Search Console — first-time setup

### Step 1 — generate verification code

1. Log in to [search.google.com/search-console](https://search.google.com/search-console)
   with `apple.info.9124@gmail.com` (or whichever Google account owns
   the domain).
2. Click **Add property** → choose **URL prefix** → enter
   `https://sericia.com` → Continue.
3. Google offers 5 verification methods. Pick **HTML tag**.
4. Copy the value attribute from the meta tag — looks like:

   ```html
   <meta name="google-site-verification" content="ABCdef123…" />
   ```

### Step 2 — paste into Coolify env

1. https://coolify.{domain} → Sericia storefront → **Environment Variables**
2. Add `NEXT_PUBLIC_GSC_VERIFICATION` = `ABCdef123…` (only the content
   value — the meta tag wrapper is generated by Next.js automatically)
3. Mark as **Buildtime** AND **Runtime** (Next.js metadata is build-time
   inlined when present).
4. **Save** → **Redeploy** the storefront.

### Step 3 — verify in Google Search Console

1. Wait ~1 minute for deploy.
2. Back in Search Console → click **Verify** on the verification screen.
3. Google fetches `https://sericia.com/`, finds the meta tag, marks
   the property as verified.

### Step 4 — submit sitemap

1. In Search Console → **Sitemaps** (left sidebar).
2. Enter `sitemap.xml` and click **Submit**.
3. Google starts crawling within minutes. First indexed pages appear
   in the **Pages** report within 24h–14 days.

### Step 5 — Bing Webmaster Tools (parallel, optional)

Bing has a separate console:
- https://www.bing.com/webmasters
- Same flow: HTML tag → `NEXT_PUBLIC_BING_VERIFICATION` env →
  redeploy → verify → submit sitemap.
- Bonus: Bing also accepts the IndexNow API submissions covered
  in §3 below, so even without manual sitemap submission your URLs
  flow into Bing's index automatically.

### Step 6 — Yandex Webmaster (low priority)

Only worth setting up if Russian / CIS market matters short-term:
- https://webmaster.yandex.com
- Same flow: HTML tag → `NEXT_PUBLIC_YANDEX_VERIFICATION` env →
  redeploy → verify → submit sitemap.

## 3. IndexNow — Bing/Yandex/Naver instant submission

The `/api/indexnow` endpoint pings IndexNow protocol partners with
URLs you've just published. Bing, Yandex, Naver, Seznam, and Yep all
participate.

### Manual single-URL submission

```bash
curl -X POST https://sericia.com/api/indexnow \
  -H "Content-Type: application/json" \
  -H "X-Admin-Secret: $SERICIA_ADMIN_SECRET" \
  -d '{"urls":["https://sericia.com/products/sencha"]}'
```

Expected: `HTTP 200 {"ok":true,"submitted":1,"status":200,"note":"Accepted"}`

### Bulk submission (after major content drops)

Run the `indexnow-bulk-submit.ts` script — it reads `/sitemap.xml`,
extracts every URL, and submits them in batches of 500 with a 2-second
spacing between batches:

```bash
SERICIA_ADMIN_SECRET=xxx \
  npx tsx storefront/scripts/indexnow-bulk-submit.ts
```

Use cases:

- **First-launch flood**: pre-warm Bing / Yandex / Naver with the full
  ~3,300-URL catalogue right after deploy. Index in hours, not weeks.
- **Post-pSEO-expand catch-up**: after running the pSEO matrix
  generator (§5), bulk-submit again.
- **Monthly nudge**: cron to re-ping the search engines so deleted /
  updated URLs propagate quickly.

### Auto-ping on Payload publish (optional, recommended)

Add a Payload `afterChange` hook on the Articles collection that posts
the article URL to `/api/indexnow` whenever an editor publishes a
piece. Implementation TODO — see `collections/Articles.ts`.

### IndexNow key file

The key proves ownership. It lives at:

```
https://sericia.com/f55bbec16474b7c82fe2582eb4d349be.txt
```

Don't delete that file. If you rotate the key, update both
`/api/indexnow/route.ts` (`INDEXNOW_KEY` constant) and the
`public/<key>.txt` filename.

## 4. Sitemap submission flow

### Where the sitemap lives

`https://sericia.com/sitemap.xml` — generated dynamically by Next.js
from `app/(frontend)/sitemap.ts`. The route is cached for 1 hour by
Cloudflare per the `Storefront HTML — 1h edge cache + SWR` rule.

### Forcing a sitemap refresh

When you add new routes (e.g. expand the pSEO matrix, add new journal
articles), purge Cloudflare's cached sitemap:

```bash
curl -X POST "https://api.cloudflare.com/client/v4/zones/938da62e2b061dcde614c314da72fcb6/purge_cache" \
  -H "Authorization: Bearer $CLOUDFLARE_API_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"files":["https://sericia.com/sitemap.xml"]}'
```

(Token needs `Zone.Cache Purge:Edit` — the cache-rules-only token is
NOT sufficient. Generate a new scoped token if needed.)

### Re-submitting after major changes

1. Force-refresh the sitemap cache (above).
2. In Google Search Console → Sitemaps → click the sitemap row →
   **Resubmit**. Optional but it nudges Google's recrawl.
3. Run `indexnow-bulk-submit.ts` to push the same URLs to Bing/Yandex.

## 5. pSEO matrix — running the bulk generator

The pSEO infrastructure (T3-C from F35) generates content for every
matrix cell using DeepSeek V3 with Context Caching. Cost projection
at full F40 matrix: **≈ $5.64 lifetime** (one-time cache warm).

### Step 1 — seed the brief queue

```bash
# Seed canonical /guides/[country]/[product] briefs (144 base × 10 locales = 1,440)
NEXT_PUBLIC_SUPABASE_URL=… SUPABASE_SERVICE_ROLE_KEY=… \
  npx tsx storefront/scripts/expand-pseo-briefs.ts
```

This reads the matrix from `lib/pseo-matrix.ts` and bulk-inserts into
the Supabase `sericia_pseo_briefs` queue. Idempotent — partial unique
index on `(lower(topic), locale)` skips already-pending/done rows.

### Step 2 — run the queue drainer

```bash
SERICIA_ADMIN_SECRET=… \
  npx tsx storefront/scripts/drain-pseo-queue.ts
```

Sequential (DeepSeek V3 cache discount only applies on serial calls
with byte-identical prefix). Approx 25 articles/minute at ~$0.014/M
input. Full 1,440-article run ≈ 60 minutes, $1.28.

### Step 3 — submit the new URLs to IndexNow

After the drainer finishes:

```bash
SERICIA_ADMIN_SECRET=… \
  npx tsx storefront/scripts/indexnow-bulk-submit.ts
```

Bing/Yandex/Naver get the full sitemap including the freshly-generated
guides within minutes.

### Step 4 — verify ranking lift

Wait 7–14 days, then check Google Search Console → Performance →
Filter by `URL contains: /guides/`. Expect:

- First impressions appear within 3–7 days
- First clicks within 7–14 days
- Cumulative clicks ramp slowly — pSEO is a long-tail strategy

## 6. Content quality checklist (GEO + traditional SEO)

Every page (storefront, journal, guide, compare, uses, tool) should
satisfy this checklist before submission. The pSEO generator's prompt
already enforces most of it — these are the human-review gates.

### GEO (AI search) requirements
- [ ] **TL;DR first** — opening paragraph summarises the answer in
      ≤3 sentences. Perplexity / ChatGPT / Gemini extract from here.
- [ ] **Definitive language** — "X is Y" not "X might be Y".
- [ ] **Statistics with source attribution** — e.g. "Sericia ships
      to 23+ countries via EMS." Numbers are AI-citation gold.
- [ ] **FAQPage JSON-LD** — at least 4 Q&A pairs. Already standard
      on `/faq` and the pSEO `/guides/*` pages.

### Traditional SEO requirements
- [ ] **H1 contains primary keyword** — exactly once per page.
- [ ] **H2 = question form** — "What is sencha?", "How to brew matcha?"
- [ ] **Internal links** — every page links to ≥3 related pages.
      pSEO matrix routes auto-generate these grids.
- [ ] **Image alt text** — every `<img>` (and `next/image`) has alt.
- [ ] **Word count** — pillar pages 1,500–3,000 words, cluster pages
      800–1,500 words. Tools / legal pages exempt.

### Schema.org coverage
- [ ] **BreadcrumbList** on every nested route (auto via shared util)
- [ ] **Article** schema on journal / guides / compare / uses
- [ ] **Product** schema on `/products/[slug]`
- [ ] **Recipe** schema on tools that take ingredients (e.g. dashi-ratio)
- [ ] **FAQPage** schema on every page that has Q&A content

## 7. Monitoring + re-submission cadence

| Cadence | Action |
|---------|--------|
| **On every deploy** | F40 added a Coolify post-deploy hook that automatically pings IndexNow with the changed URLs. (TODO — file not yet created, manual run for now) |
| **Daily** | Check Google Search Console → Pages → Filter by status:Excluded. Investigate any "Crawled — currently not indexed" patterns. |
| **Weekly** | Re-run `indexnow-bulk-submit.ts` (Bing's index is sticky but a weekly refresh hedges against page deletions). |
| **Monthly** | Audit content quality checklist (§6) on a 10-page sample. Re-generate stale pSEO briefs if Search Console flags them as low-quality. |
| **Quarterly** | Review hreflang coverage (10 locales × all dynamic routes). Add missing locale variants if traffic data justifies. |
